# 目的
`$ARGUMENTS` のリポジトリを外部から調査し、日本語で分かりやすくまとめる。

# 対象スコープ（read-only）
- リポジトリ URL: `$ARGUMENTS`
- README / CONTRIBUTING / docs などの主要ドキュメント
- ルートディレクトリ構成と主要設定ファイル（package.json, pyproject.toml, Cargo.toml, go.mod, main.tf 等）
- エントリポイントとなる代表的なソースファイル

# 実施内容
1. リポジトリ全体構成を把握
2. 以下を調査して整理：
   - **アーキテクチャ概要**（モノリス/マイクロサービス、レイヤ構成など）
   - **技術スタック**（言語・フレームワーク・主要ライブラリ）
   - **エントリポイント**（CLI / サーバー起動箇所 / main 関数など）
   - **セットアップ手順**（README ベース、なければ推測）
   - **主要ディレクトリの役割**
3. 設計上の特徴や気づいた点を補足

# 制約
- 一切の変更は行わない（read-only 調査）
- 推測箇所には「※推測」と明記
- 日本語で記述、サービス名には英語併記を適宜

# 期待する出力形式

```markdown
## リポジトリ概要
<1〜2段落>

## アーキテクチャ
<構成図や主要レイヤの説明>

## 技術スタック
- **言語**:
- **主要フレームワーク**:
- **主要ライブラリ**:
- **インフラ / ビルドツール**:

## エントリポイント
<起動方法・主要ファイル>

## ディレクトリ構成
<主要ディレクトリと役割>

## セットアップ手順
<手順箇条書き>

## 補足・所感
<設計上の特徴、気になる点>
```
