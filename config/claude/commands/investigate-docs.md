# 目的
`$ARGUMENTS` のドキュメントを読み込み、実装に必要な知識を日本語で整理する。
引数にフォーカストピックが含まれる場合はそのトピック中心に、無ければ全体像中心にまとめる。

# 対象スコープ（read-only）
- ドキュメントサイトのトップ URL: `$ARGUMENTS` の先頭 URL
- そのサイトのトップページ、Getting Started、主要概念ページ、API リファレンスの代表的な項目
- フォーカストピック指定がある場合は、そのトピックに関連する複数ページを横断的に参照

# 実施内容
1. トップページから全体構成を把握
2. 以下を調査して整理：
   - **このツール / ライブラリの目的**（何を解決するか）
   - **主要な概念・用語**（専門用語の日本語解説付き）
   - **インストール / セットアップ手順**
   - **代表的な使い方**(最小サンプルコード付き)
   - **主要 API / コマンド一覧**(引数・戻り値の要点)
   - **設定オプション**(よく使うもの中心)
   - **ベストプラクティスと落とし穴**
3. フォーカストピック指定がある場合は、そのトピックに該当する応用例・詳細設定を重点的に深掘り

# 制約
- 一切のコード変更・ファイル作成は行わない（read-only 調査）
- 推測箇所には「※推測」と明記
- コードサンプルは公式ドキュメントからの引用を明示（出典 URL 付き）
- 日本語で記述、技術用語には英語併記
- 取得できなかった情報は「取得不可」と明記し、捏造しない

# 期待する出力形式

```markdown
## 概要
<1〜2段落: 何のツールか、何を解決するか>

## 主要な概念
- **<用語1>**: <説明>
- **<用語2>**: <説明>

## セットアップ
<インストール手順・前提条件>

## 代表的な使い方
<最小動作サンプル（コードブロック）>

## 主要 API / コマンド
| 名前 | 役割 | 主な引数 |
|---|---|---|
| ... | ... | ... |

## 設定オプション
<よく使う設定項目>

## ベストプラクティスと落とし穴
- <Good practice 1>
- <Gotcha 1>

## 参照 URL
- <引用元URL1>
- <引用元URL2>

## (フォーカストピックがある場合) <トピック名> の詳細
<深掘り内容>
```
